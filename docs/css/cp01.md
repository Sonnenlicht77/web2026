# 📖 第1章：CSS 的三种存在形态与层叠规则

## 1.1 浏览器默认样式、用户样式、作者样式 🧬

CSS 并非只有我们写的 `.css` 文件。浏览器在渲染页面时，会合并以下**三个层级的样式**，最终决定元素的呈现。

### 1️⃣ 浏览器默认样式（User-Agent Stylesheet）

每个浏览器都内置了一套默认样式，用于让没有样式的 HTML 文档具备基本的可读性。

**举例说明：**

```html
<h1>这是一个标题</h1>
<a href="#">这是一个链接</a>
<ul>
  <li>列表项</li>
</ul>
```

即使我们没有写任何 CSS，以上元素在 Chrome 中也会展示出特定的样式：
- `<h1>` 字体大、加粗、上下有外边距。
- `<a>` 为蓝色、带下划线。
- `<ul>` 左侧有内边距（缩进效果）。

**浏览器默认样式实际内容（Chrome 部分节选）：**

```css
/* 来自 Chrome 的 html.css 内部样式表 */
h1 {
  display: block;
  font-size: 2em;
  margin-block-start: 0.67em;
  margin-block-end: 0.67em;
  font-weight: bold;
}
a:-webkit-any-link {
  color: -webkit-link;
  text-decoration: underline;
  cursor: pointer;
}
ul {
  display: block;
  list-style-type: disc;
  margin-block-start: 1em;
  margin-block-end: 1em;
  padding-inline-start: 40px;
}
```

**这就是为什么我们通常要引入 `reset.css` 或 `normalize.css`——为了统一不同浏览器之间的默认样式差异。**

---

### 2️⃣ 用户样式（User Stylesheet）

用户（即网站访问者）可以通过浏览器设置自定义样式，覆盖网站原本的样式。虽然极少普通用户会使用，但在无障碍访问场景下非常重要。

**举例说明：**

一位视力障碍用户可以在 Chrome 中安装「Stylus」扩展或直接在 `chrome://settings/appearance` 中强制设置网页最小字号为 `20px`，或自定义颜色对比度。

用户样式表片段：

```css
/* 用户通过扩展注入的样式 */
body {
  font-size: 20px !important; /* 用户强行放大文字 */
  background-color: black !important;
  color: yellow !important;
}
```

**在层叠规则中，用户样式表的 `!important` 声明拥有极高的优先级**，甚至能覆盖作者样式中的 `!important`（见下一节详解）。这是为了确保用户的自定义需求不被网站样式绑架。

---

### 3️⃣ 作者样式（Author Stylesheet）

这就是我们前端开发者日常编写的 CSS——通过 `<link>` 引入的外部样式表、`<style>` 标签内的内部样式表、或元素上的 `style` 行内样式。

**举例说明：**

```css
/* style.css - 作者样式 */
h1 {
  color: red;
  font-size: 24px;
}
```

**优先级图示（无 `!important` 情况）：**
```
作者样式 (Author)  >  用户样式 (User)  >  浏览器默认样式 (User-Agent)
```
但当引入 `!important` 时，顺序会发生逆转，具体请看 1.2 节。

---

## 1.2 核心世界观：层叠（Cascade）的权重算法与 `!important` 的禁区 ⚖️

层叠是 CSS 的 C（Cascading），它决定了当多条规则同时作用于同一元素时，**哪一条规则胜出**。

### 权重计算步骤（由高到低排列）

当浏览器比较两条冲突的规则时，按以下**优先级顺序**裁决，一旦某一级分出胜负，则立即停止比较：

| 层级 | 规则来源与重要性 | 权重级别（1 最高） |
|------|-----------------|-------------------|
| 1 | **Transition 过渡声明**（动画进行中） | 最高临时权重 |
| 2 | 浏览器默认样式中的 `!important` | 极高 |
| 3 | 用户样式中的 `!important` | 非常高 |
| 4 | **作者样式中的 `!important`** | 高 |
| 5 | **CSS 动画 `@keyframes`** | 较高 |
| 6 | **作者样式（普通声明）** | 中 |
| 7 | **用户样式（普通声明）** | 低 |
| 8 | **浏览器默认样式（普通声明）** | 最低 |

---

### 详细实例：`!important` 为什么危险且需要规避？

考虑以下代码：

```html
<p id="intro" class="text" style="color: green;">Hello CSS</p>
```

```css
/* 作者样式 - 外部样式表 */
p {
  color: blue !important;
}
#intro {
  color: red;
}
.text {
  color: orange;
}
```

**问：最终文字是什么颜色？**

**答：蓝色**。因为虽然行内样式 `style="color: green;"` 的特异性非常高，但作者样式中的 `!important` 会直接跃升到权重表的第 4 级，高于普通作者样式（第 6 级）和行内样式（也属于作者样式的一部分，特异值极高但未使用 `!important`）。

**⚠️ 禁区警示：**

1. **一旦使用 `!important`，后续维护性急剧下降**：如果想要覆盖这个蓝色，只能使用更高层级的 `!important`（如用户样式 `!important` 或更高特异性的作者 `!important`），形成恶性循环。
2. **大型项目中 `!important` 通常是设计系统崩溃的信号**。建议仅在以下极端场景使用：
   - 覆盖第三方库中无法通过选择器特异性解决的侵入式样式。
   - 工具类辅助样式（如 `.hidden { display: none !important; }`）。

---

### 权重对比实战代码

```html
<div class="box" id="box" style="background-color: yellow;">Box</div>
```

```css
/* 规则 A：元素选择器 */
div {
  background-color: gray; /* 特异性 (0,0,1) */
}

/* 规则 B：类选择器 */
.box {
  background-color: blue; /* 特异性 (0,1,0) */
}

/* 规则 C：ID 选择器 */
#box {
  background-color: red; /* 特异性 (1,0,0) */
}

/* 规则 D：作者 !important */
div {
  background-color: purple !important; /* 越过特异性，直接靠来源重要性取胜 */
}
```

**层叠结果分析：**

- 如果没有 `!important`：行内样式（特异值 `1,0,0,0`）胜出 → **黄色**。
- 加上规则 D 后：作者 `!important` 优先级碾压行内样式 → **紫色**。

### 🔬 特别加餐：层叠体系的活样本 —— 过渡动画与权重的精确关系  

**事实上，过渡是否平滑与 `!important` 毫无关系，它只关心属性值的“变化”本身。**

#### ✅ 实验一：无 `!important` 的标准过渡

```html
<style>
  .box {
    width: 100px;
    height: 100px;
    background-color: blue;          /* 初始值：层级 6（作者普通声明） */
    transition: background-color 3s; /* 过渡时长 3 秒，便于观察 */
  }
  .box:hover {
    background-color: red;           /* 终点值：层级 6（作者普通声明） */
  }
</style>
<div class="box"></div>
```

**现象**：悬停在蓝色方块上，背景色会在 3 秒内从蓝色平滑过渡到红色。

**层叠解析**：
- 悬停前，`.box` 的 `blue` 生效。
- 悬停瞬间，`.box:hover` 的 `red` 与 `blue` 冲突。由于两者同为作者普通声明（层级 6），且 `.box:hover` 特异性更高（`0,0,2,0` vs `0,0,1,0`），故 `red` 胜出。
- 由于存在 `transition` 声明，浏览器不会瞬间切换，而是启动一个 3 秒的插值过程。**在此期间，浏览器持续生成临时的 `background-color` 内联样式（例如 `#7722aa`、`#b23c3c` 等中间值）。**
- 这个临时内联样式属于**层级 1（过渡临时声明）**，它会在整个过渡过程中**覆盖掉所有层级 6 的普通作者声明**（包括 `.box` 的蓝色和 `.box:hover` 的红色）。

**DevTools 验证**：
1. 打开 Chrome DevTools（F12），切换到 **Elements** 面板。
2. 选中 `<div class="box">`，右侧 **Styles** 标签页。
3. 用鼠标悬停在方块上**但不移开**，观察 `element.style` 栏。
4. 你会看到一个类似 `background-color: #8822aa;` 的条目，它**前面没有选择器**，并且它会**划掉**下方 `.box` 和 `.box:hover` 中的颜色声明。
5. 当过渡结束（3 秒后），该临时内联样式消失，最终由 `.box:hover` 的 `red` 接管。

---

#### ⚠️ 实验二：终点值带有 `!important` 的过渡

```html
<style>
  .box {
    width: 100px;
    height: 100px;
    background-color: blue;          /* 层级 6：作者普通声明 */
    transition: background-color 3s;
  }
  .box:hover {
    background-color: red !important; /* 层级 4：作者 !important */
  }
</style>
<div class="box"></div>
```

**现象**：悬停时，背景色依然会平滑过渡到红色，与实验一视觉上完全一致。

**关键区别**：
- 悬停瞬间，`.box:hover` 的 `red !important` 属于**层级 4**，而 `.box` 的 `blue` 属于层级 6，显然 `red` 胜出。
- 浏览器同样启动插值过程，生成临时内联样式。**重点来了：这个临时内联样式的权重，会“继承”冲突双方中较高层级的权重。**
- 由于终点值属于层级 4，因此过渡中间值也被视为**层级 1 但带有层级 4 优先级的临时声明**。它不仅能覆盖所有层级 6 的普通声明，甚至能覆盖层级 5 的 `@keyframes` 动画声明（假如同时存在的话）。
- 过渡结束后，层级 4 的 `red !important` 正式生效。

**结论**：`!important` 不会阻碍过渡的发生，它只决定**哪个颜色值最终参与插值计算**。过渡的平滑效果是由 `transition` 属性独立控制的。

---

#### 🧪 进阶实验：用行内样式验证临时内联的统治力

为了更直观地展示“层级 1 在过渡期间是王”，这里设计一个行内样式干扰的极端场景：

```html
<style>
  .box {
    width: 100px;
    height: 100px;
    background-color: blue;
    transition: background-color 3s;
  }
  /* 一个永远存在的高特异性干扰规则（无 !important） */
  #box.box {
    background-color: green; /* 特异性 (0,1,1,0)，远高于单类 */
  }
  .box:hover {
    background-color: red;
  }
</style>

<!-- 行内样式：特异性最高 (1,0,0,0)，但仍是普通声明（层级 6） -->
<div class="box" id="box" style="background-color: yellow;"></div>
```

**现象与层叠推演**：
- **悬停前**：行内样式 `yellow` 特异性最高，胜出。`#box.box` 的 `green` 被划掉，`.box` 的 `blue` 更不用说。
- **悬停瞬间**：`.box:hover` 的 `red` 也是普通声明（层级 6），虽然其特异性只有 `(0,0,2,0)`，但因为**来源顺序靠后且同为层级 6**，它战胜了行内样式的 `yellow`（同样层级 6，来源顺序靠前）。此时如果没有过渡，颜色会瞬间变红。
- **过渡期间**：浏览器生成临时内联中间色（例如橙色）。这个临时内联属于**层级 1**，它会霸道地**划掉一切**——包括行内样式 `yellow`、`#box.box` 的 `green`、`.box:hover` 的 `red` 以及 `.box` 的 `blue`。在 DevTools 中你将看到一条无选择器的 `background-color` 正在统治一切。

---

#### 📌 最终修正后的层叠层级速记表（含过渡细节）

| 层级 | 规则来源 | 与过渡的关系说明 |
|------|----------|------------------|
| 1 | **过渡临时内联声明** | 过渡进行时动态生成。若终点值为普通声明，临时声明覆盖层级 5~8；若终点值为 `!important`，临时声明继承该权重，覆盖层级 4 以下的普通声明。 |
| 2 | 浏览器默认样式中的 `!important` | 过渡无法覆盖此层。若 UA 样式使用 `!important`，过渡对相关属性无效。 |
| 3 | 用户样式中的 `!important` | 同上，过渡无法覆盖用户 `!important`。 |
| 4 | **作者样式中的 `!important`** | 若过渡的目标值属于该层，过渡中间值也具备该层权重，从而覆盖层级 5 及以下的普通声明。 |
| 5 | CSS 动画 `@keyframes` | 动画与过渡是两种不同的动态样式生成机制。若同时作用于同一属性，**动画（层级 5）的每一帧都会被过渡（层级 1）的临时内联覆盖**（只要过渡正在运行）。 |
| 6 | 作者样式（普通声明） | 过渡中间值可以覆盖该层所有声明（包括行内样式）。 |
| 7 | 用户样式（普通声明） | 会被过渡中间值覆盖。 |
| 8 | 浏览器默认样式（普通声明） | 会被过渡中间值覆盖。 |

---

#### 💎 核心结论

1. **过渡的平滑效果与目标值是否带 `!important` 无关**，它只要求属性值发生改变。
2. **过渡期间浏览器生成的临时内联样式（层级 1）是压倒性的**，它会覆盖所有普通来源声明，甚至在某些情况下压制动画。
3. **使用 DevTools 观察 `element.style` 栏的动态变化**，是理解层叠权重最直观的方式。
4. **层叠规则不是纸上谈兵**，在动画与过渡的交织中，它们生动地展现着 CSS 的设计哲学。

## 1.3 来源顺序、特异性（Specificity）计算题实战 🎯

### 特异性计算规则（a, b, c, d 计数法）

- **a**：行内样式（`style="..."`），值为 1 或 0。
- **b**：ID 选择器的数量。
- **c**：类选择器、属性选择器、伪类（如 `:hover`）的数量。
- **d**：元素选择器、伪元素（如 `::before`）的数量。

**通配符 `*`、组合器（`>`、`+`、`~`）、`:where()` 对特异性无贡献（计为 0）。**

---

### 计算题实战

**题目 1：**

```css
/* 规则 1 */
div ul li.active a {
  color: red;
}
/* 计算：元素(4) + 类(1) → (0,0,1,4) */

/* 规则 2 */
#sidebar .nav-link:hover {
  color: blue;
}
/* 计算：ID(1) + 类(1) + 伪类(1) + 元素(0) → (0,1,2,0) */

/* 规则 3 */
body #content .post p:first-child {
  color: green;
}
/* 计算：ID(1) + 类(1) + 伪类(1) + 元素(2: body, p) → (0,1,2,2) */
```

**比较：** (0,1,2,2) > (0,1,2,0) > (0,0,1,4)，所以**规则 3 胜出**。

---

**题目 2：陷阱题——`:not()` 和 `:is()` 的特异性**

```css
/* 规则 A */
a:not(.exclude) {
  color: black;
}
/* 特异性等于 :not() 内部选择器的最高特异性 → (0,0,1,1) */

/* 规则 B */
a.link {
  color: orange;
}
/* 特异性：类(1) + 元素(1) → (0,0,1,1) */

/* 两者特异性相同，此时看来源顺序，后出现的胜出 → 橙色 */
```

---

**题目 3：`:where()` 的零特异性魔法**

```css
/* 规则 X */
:where(#header) .title {
  color: red;
}
/* :where() 内部无论写什么特异性都是 0 → 最终 (0,0,1,0) */

/* 规则 Y */
.title {
  color: blue;
}
/* 最终 (0,0,1,0) 与 X 相同，靠顺序胜出 */
```

**:where() 非常适用于建立基础样式库，方便后续轻松覆盖，无需提高特异性。**

## 1.4 现代调试技巧：Chrome DevTools 中查看层叠层与特异性 🛠️

### 场景模拟

假设我们有如下混乱的样式表：

```html
<style>
  /* 作者样式 */
  .title {
    color: green !important;
  }
  #main h1 {
    color: red;
  }
  h1 {
    color: blue;
  }
</style>

<h1 id="main-heading" class="title" style="color: purple;">Hello Debugging</h1>
```

#### DevTools 调试步骤（以 Chrome 为例）

**1. 检查元素，查看「Styles」面板**

右键点击 `<h1>` → **检查**，右侧 Styles 面板会显示所有作用于该元素的规则，并按权重排序。

你会看到类似以下的展示（从上到下代表胜出者）：

```
▼ element.style {
    color: purple;   /* 被划掉（删除线） */
  }

▼ .title {
    color: green !important;  /* 胜出，未被划掉 */
  }

▼ #main h1 {
    color: red;      /* 被划掉 */
  }

▼ h1 {
    color: blue;     /* 被划掉 */
  }
```

**2. 查看特异性和被覆盖原因**

将鼠标悬停在**被划掉**的 `color: purple` 上，DevTools 会弹出提示框：

> `color: purple` is overridden by `.title` with `color: green !important`.

如果是普通覆盖（无 `!important`），它会显示：
> `color: red` was overridden by `#main-heading` (specificity: 1,0,0).

**3. 使用「Computed」面板追溯胜出值**

切换到 **Computed** 子标签页，找到 `color` 属性，点击左侧的小箭头展开，会显示：

- **Winning declaration**: `.title` — `color: green !important`
- **Overridden declarations**: 列出所有被淘汰的规则及其失败原因。

**4. 查看层叠层（`@layer`）的影响（现代浏览器）**

如果你的项目使用了 `@layer` 组织样式，Styles 面板会额外显示该规则所在的图层名称，例如：

```
▼ @layer components {
    .title { color: green !important; }
  }
```

点击图层名，可以跳转到 Sources 面板定位源码。

---

### 高级技巧：审查用户代理样式

有时你想知道某个默认样式是从哪来的，比如 `<input>` 为什么有奇怪的边框。

在 Computed 面板中展开对应属性，若来源显示为 `user agent stylesheet`，点击右侧的文件链接（如 `html.css`），可以直接查看浏览器的内部样式表源代码。

---

**本章小结**：理解 CSS 的三级存在形态与层叠权重算法，是解决 90% 样式冲突问题的基石。配合 Chrome DevTools 的特异性可视化工具，你可以从「盲目加 `!important`」进化为「精准计算权重」的 CSS 架构师。
