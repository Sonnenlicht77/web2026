# 🤸 第5章：Flexbox —— 一维弹性布局完全指南

如果说文档流是 CSS 布局的「自然规律」，那么 `Flexbox` 就是工程师手中的「精密工具」。它专为**一维布局**而生——要么是水平的一行，要么是垂直的一列。掌握 `Flexbox`，意味着你能轻松应对**导航栏、卡片列表、垂直居中、等分布局**等日常开发中的绝大部分排版需求。



## 📏 5.1 主轴与交叉轴：基础轴线概念的彻底可视化

`Flexbox` 的核心设计基于两条轴线：**主轴（Main Axis）** 和 **交叉轴（Cross Axis）**。几乎所有的 `Flex` 属性，都在围绕这两条轴定义元素如何排列、对齐和伸缩。

### 🧭 轴线的方向由 `flex-direction` 决定

| `flex-direction` 值 | 主轴方向 | 交叉轴方向 |
|---------------------|----------|------------|
| `row`（默认） | 水平从左到右 → | 垂直从上到下 ↓ |
| `row-reverse` | 水平从右到左 ← | 垂直从上到下 ↓ |
| `column` | 垂直从上到下 ↓ | 水平从左到右 → |
| `column-reverse` | 垂直从下到上 ↑ | 水平从左到右 → |

**可视化代码示例**：

::: code-group

```css
.flex-demo {
  display: flex;
  flex-direction: row;
  border: 2px solid #3498db;
  padding: 10px;
}
.flex-demo .item {
  width: 50px;
  height: 50px;
  background: #aed6f1;
  border: 1px solid #2980b9;
  display: flex;
  align-items: center;
  justify-content: center;
  font-weight: bold;
}
```

```html
<div class="flex-demo">
  <div class="item">1</div>
  <div class="item">2</div>
  <div class="item">3</div>
</div>
<p>👆 flex-direction: row —— 主轴水平，交叉轴垂直</p>
```

:::



**核心认知**：
- **主轴方向**：元素排列的**流向**。`justify-content` 沿主轴控制元素**之间的间距与对齐**。
- **交叉轴方向**：与主轴垂直。`align-items` 沿交叉轴控制元素在**行内/列内的对齐方式**。

**改变方向的对比实验**：


::: code-group

```css
.demo-container {
  display: flex;
  gap: 20px;
}
.flex-col {
  display: flex;
  flex-direction: column;
  border: 2px solid #e74c3c;
  padding: 10px;
  height: 150px;
}

```

```html
<div class="demo-container">
  <div class="flex-col">
    <div class="item">A</div>
    <div class="item">B</div>
    <div class="item">C</div>
  </div>
  <p>👆 flex-direction: column<br>主轴垂直 ↓，交叉轴水平 →</p>
</div>
```

:::

**关键启示**：当你将 `flex-direction` 切换为 `column` 时，`justify-content` 将控制**垂直方向**的间距，`align-items` 则控制**水平方向**的对齐。这是许多开发者最初混淆的地方。



## ⚙️ 5.2 容器属性详解：`justify-content`、`align-items` 与 `gap` 的黄金组合

Flex 容器上有六个核心属性，其中最常用的黄金三角是 `justify-content`、`align-items` 和 `gap`。它们共同决定了子元素如何分布与对齐。

### 1️⃣ `justify-content`：沿主轴的对齐与分布

```css
.container {
  display: flex;
  justify-content: flex-start | flex-end | center | space-between | space-around | space-evenly;
}
```

**可视化对比**（假设有四个子项）：

| 值 | 效果描述 | 适用场景 |
|----|----------|----------|
| `flex-start` | 子项紧靠主轴起点 | 默认左对齐 |
| `flex-end` | 子项紧靠主轴终点 | 右对齐的导航菜单 |
| `center` | 子项居中，两侧无额外空间 | 居中 logo + 导航 |
| `space-between` | 首尾贴边，中间均分 | 卡片头部（标题靠左，按钮靠右） |
| `space-around` | 每个子项两侧间距相等（首尾是中间的一半） | 标签云 |
| `space-evenly` | 所有间隙完全相等（包括首尾与边缘） | 底部 Tab 栏 |

**实战演示**：

```html
<style>
  .justify-demo {
    display: flex;
    justify-content: space-between;
    background: #f4f6f7;
    padding: 10px;
    border-radius: 8px;
  }
</style>
<div class="justify-demo">
  <span>🍔 汉堡</span>
  <span>🍕 披萨</span>
  <span>🌮 塔可</span>
</div>
```

**效果**：三个食物图标分别位于容器的左、中、右位置，且间距自动计算。

### 2️⃣ `align-items`：沿交叉轴的对齐方式

```css
.container {
  display: flex;
  align-items: stretch | flex-start | flex-end | center | baseline;
}
```

| 值 | 效果描述 | 典型用途 |
|----|----------|----------|
| `stretch`（默认） | 子项在交叉轴方向拉伸填满容器高度（若未设置固定高） | 等高列布局 |
| `flex-start` | 子项顶部对齐 | 顶部对齐的卡片 |
| `flex-end` | 子项底部对齐 | 底部按钮对齐 |
| `center` | 子项垂直居中 | **垂直居中的万能方案** |
| `baseline` | 子项内文字基线对齐 | 不同字号文字对齐 |

**垂直居中的经典一行**：

```css
.parent {
  display: flex;
  align-items: center;  /* 垂直居中 */
  justify-content: center; /* 水平居中 */
  height: 200px;
}
```

从此告别 `line-height`、`position`、`transform` 的繁琐组合。

### 3️⃣ `gap`：行列间隙的优雅设置

`gap` 属性是 Flexbox 中定义子项之间间距的现代方案，替代了繁琐的 `margin` 计算。

```css
.container {
  display: flex;
  gap: 20px;           /* 同时控制行间距和列间距（单行情况下即为子项间距） */
  /* 或分开设置 */
  row-gap: 10px;
  column-gap: 30px;
}
```

**对比**：传统方法需要给每个子项加 `margin-right`，然后用 `:last-child` 移除最后一个的边距。`gap` 彻底终结了这种「边距地狱」。

**实际应用**：

```html
<style>
  .tag-list {
    display: flex;
    flex-wrap: wrap;
    gap: 8px 12px;  /* 行间距8px，列间距12px */
  }
  .tag {
    padding: 4px 12px;
    background: #e8f0fe;
    border-radius: 16px;
  }
</style>
<div class="tag-list">
  <span class="tag">CSS</span>
  <span class="tag">JavaScript</span>
  <span class="tag">React</span>
  <span class="tag">Flexbox</span>
  <span class="tag">响应式</span>
</div>
```

### 4️⃣ 其他容器属性速览

| 属性 | 作用 | 常用值 |
|------|------|--------|
| `flex-wrap` | 是否允许换行 | `nowrap`（默认）、`wrap`、`wrap-reverse` |
| `flex-flow` | `flex-direction` 和 `flex-wrap` 的简写 | `row wrap` |
| `align-content` | 多行/多列时，行/列在交叉轴上的分布 | `space-between`、`center`（类似 `justify-content` 但作用于交叉轴） |



## 🎈 5.3 项目属性实战：`flex-grow` 挤压算法与 `flex-shrink` 的收缩抵抗

Flex 容器内的子项（项目）可以通过三个核心属性控制自己的伸缩行为：`flex-grow`、`flex-shrink`、`flex-basis`。它们通常通过简写属性 `flex` 统一设置。

### 📈 `flex-grow`：瓜分剩余空间的贪婪指数

- **默认值**：`0`，不参与瓜分。
- **规则**：将容器主轴上的**剩余空间**按 `flex-grow` 的比例分配给各项目,考虑文本内容宽度。

**示例**：

```html
<style>
  .grow-container {
    display: flex;
    width: 400px;
    border: 2px solid #2ecc71;
  }
  .grow-item {
    height: 60px;
  }
  .grow-1 { flex-grow: 1; background: #a9dfbf; }
  .grow-2 { flex-grow: 2; background: #7dcea0; }
  .grow-0 { flex-grow: 0; width: 80px; background: #f9e79f; }
</style>
<div class="grow-container">
  <div class="grow-item grow-1">flex-grow: 1</div>
  <div class="grow-item grow-2">flex-grow: 2</div>
  <div class="grow-item grow-0">固定 80px</div>
</div>
```

**计算过程**：
- 容器宽度：400px
- 固定项目：80px
- 剩余空间：400 - 80 = 320px
- `grow` 总和：1 + 2 = 3
- 项目1获得：320 × (1/3) ≈ 106.67px
- 项目2获得：320 × (2/3) ≈ 213.33px


**实际应用**：经典的「左侧固定，右侧自适应」两栏布局：

```css
.sidebar {
  width: 250px;
}
.main {
  flex-grow: 1; /* 吃掉所有剩余宽度 */
}
```

### 📉 `flex-shrink`：空间不足时的收缩抵抗

- **默认值**：`1`，允许收缩。
- **规则**：当容器空间不足以容纳所有项目的 `flex-basis`（或 `width`）时，按 `flex-shrink` 的比例收缩。

**示例**：容器宽度 300px，三个子项各宽 150px（总宽 450px）。

```html
<style>
  .shrink-container {
    display: flex;
    width: 300px;
    border: 2px solid #e74c3c;
  }
  .shrink-item {
    flex-basis: 150px;
    height: 60px;
  }
  .shrink-1 { flex-shrink: 1; background: #f1948a; }
  .shrink-2 { flex-shrink: 2; background: #e74c3c; color: white; }
  .shrink-0 { flex-shrink: 0; background: #f9e79f; } /* 不收缩 */
</style>
<div class="shrink-container">
  <div class="shrink-item shrink-1">shrink:1</div>
  <div class="shrink-item shrink-2">shrink:2</div>
  <div class="shrink-item shrink-0">shrink:0</div>
</div>
```

**计算逻辑**：
- 溢出空间：450 - 300 = 150px
- 参与收缩的项目只有前两个，`shrink` 权重：1×150 + 2×150 = 450
- 项目1收缩：150 × (150/450) = 50px → 最终宽度 100px
- 项目2收缩：150 × (300/450) = 100px → 最终宽度 50px
- 项目3不收缩，保持 150px（可能导致溢出或出现滚动条）

**关键技巧**：设置 `flex-shrink: 0` 可防止元素被挤压变形，常用于头像、图标等固定尺寸元素。

### 🧬 `flex` 简写：记忆口诀

```css
flex: flex-grow flex-shrink flex-basis;
/* 默认值: 0 1 auto */
```

| 简写值 | 等效完整写法 | 含义 |
|--------|-------------|------|
| `flex: 1` | `flex: 1 1 0%` | 可伸缩，平分剩余空间 |
| `flex: auto` | `flex: 1 1 auto` | 基于内容宽度伸缩 |
| `flex: none` | `flex: 0 0 auto` | 固定尺寸，不伸缩 |
| `flex: 2` | `flex: 2 1 0%` | 以 2 倍比例瓜分空间 |



## 🏛️ 5.4 经典案例：圣杯布局、等分布局、粘性页脚、自适应卡片列表

理论最终要落地。以下四个案例覆盖了 Flexbox 最常用的业务场景。

### 1️⃣ 圣杯布局（Holy Grail Layout）

**需求**：页头、页脚、中间三列（左侧导航、主内容、右侧边栏），其中中间三列等高，主内容自适应宽度。

```html
<div class="holy-grail">
  <header>Header</header>
  <div class="holy-content">
    <nav>Left Sidebar</nav>
    <main>Main Content (自适应高度和宽度)</main>
    <aside>Right Sidebar</aside>
  </div>
  <footer>Footer</footer>
</div>
```

```css
.holy-grail {
  display: flex;
  flex-direction: column;
  min-height: 100vh;           /* 全屏高度 */
}
header, footer {
  background: #34495e;
  color: white;
  padding: 15px;
  text-align: center;
}
.holy-content {
  display: flex;
  flex: 1;                    /* 撑满剩余高度 */
}
nav {
  width: 200px;
  background: #ecf0f1;
  padding: 15px;
}
main {
  flex: 1;                    /* 自适应宽度 */
  background: white;
  padding: 15px;
}
aside {
  width: 180px;
  background: #ecf0f1;
  padding: 15px;
}
footer {
  margin-top: auto;           /* 也可用 flex 实现粘性页脚 */
}
```

**要点**：
- 整体 `flex-direction: column` 实现垂直方向的弹性。
- 中间区域 `display: flex` 实现水平三列。
- `flex: 1` 让主内容自动填充剩余空间。

### 2️⃣ 等分布局（N 列等宽）

**需求**：将一行等分为 N 列，无论列数是 2、3 还是 5，每列宽度完全相等。

```html
<div class="equal-cols">
  <div class="col">列 1</div>
  <div class="col">列 2</div>
  <div class="col">列 3</div>
  <div class="col">列 4</div>
</div>
```

```css
.equal-cols {
  display: flex;
  gap: 20px;
}
.col {
  flex: 1;                    /* 每个子项等分剩余空间 */
  background: #d5e8d4;
  padding: 20px;
  text-align: center;
  border-radius: 8px;
}
```

**扩展**：若想实现「固定宽度 + 等分」混合：

```css
.col-special {
  flex: 2;   /* 这个列是其他列的 2 倍宽 */
}
```

### 3️⃣ 粘性页脚（Sticky Footer）

**需求**：当页面内容不足一屏时，页脚固定在底部；内容超过一屏时，页脚跟随内容滚动。

```html
<div class="page">
  <header>Header</header>
  <main>Main Content (内容可能很少也可能很多)</main>
  <footer>Footer</footer>
</div>
```

```css
.page {
  display: flex;
  flex-direction: column;
  min-height: 100vh;
}
main {
  flex: 1;                    
  /* 关键！main 吞掉所有剩余高度，把 footer 推到底部 */
}
```

这是 Flexbox 最简洁优雅的应用，完全取代了过去的负 `margin` 和 `calc()` 方案。

### 4️⃣ 自适应卡片列表（响应式网格）

**需求**：卡片列表，每行显示尽可能多的卡片，卡片最小宽度 250px，剩余空间自动均分。

```html
<div class="card-grid">
  <div class="card">卡片 1</div>
  <div class="card">卡片 2</div>
  <div class="card">卡片 3</div>
  <div class="card">卡片 4</div>
  <div class="card">卡片 5</div>
</div>
```

```css
.card-grid {
  display: flex;
  flex-wrap: wrap;
  gap: 20px;
}
.card {
  flex: 1 1 250px;            /* 基础宽度 250px，允许伸缩 */
  /* 等价于：
     flex-grow: 1;
     flex-shrink: 1;
     flex-basis: 250px;
  */
  background: white;
  padding: 20px;
  box-shadow: 0 2px 8px rgba(0,0,0,0.1);
  border-radius: 12px;
}
```

**效果**：
- 当容器宽度足够时，多个卡片在一行显示，并等分剩余空间。
- 当容器变窄时，一行能容纳的卡片减少，卡片宽度会略微收缩。
- 当容器宽度小于 250px 时，卡片占满整行。

这是无媒体查询实现响应式网格的经典模式。

### 5️⃣ 导航栏经典模式：Logo 靠左，菜单项靠右

```html
<nav class="navbar">
  <div class="logo">🚀 我的网站</div>
  <ul class="nav-links">
    <li><a href="#">首页</a></li>
    <li><a href="#">产品</a></li>
    <li><a href="#">关于</a></li>
  </ul>
</nav>
```

```css
.navbar {
  display: flex;
  justify-content: space-between;  /* 两端对齐 */
  align-items: center;
  padding: 0 20px;
  background: #2c3e50;
  color: white;
}
.nav-links {
  display: flex;
  gap: 30px;
  list-style: none;
}
```

### 6️⃣ 垂直居中 + 文本截断的组合

```html
<div class="message-row">
  <span class="avatar">👤</span>
  <span class="message">这是一条很长很长很长很长很长很长的消息文本，需要自动截断显示省略号</span>
  <span class="time">10:30</span>
</div>
```

```css
.message-row {
  display: flex;
  align-items: center;         /* 垂直居中 */
  gap: 12px;
  padding: 10px;
  border-bottom: 1px solid #eee;
}
.avatar {
  flex-shrink: 0;              /* 头像不收缩 */
  width: 40px;
  height: 40px;
  background: #bdc3c7;
  border-radius: 50%;
  display: flex;
  align-items: center;
  justify-content: center;
}
.message {
  flex: 1;                     /* 占据剩余空间 */
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
.time {
  flex-shrink: 0;              /* 时间不收缩 */
  color: #7f8c8d;
  font-size: 14px;
}
```

### 🎯 进阶案例：最后一行对齐问题

**🐞 问题场景**  

在使用 Flexbox 实现响应式网格时，我们通常会这样写：  

```css
.card-grid {
  display: flex;
  flex-wrap: wrap;
  justify-content: space-between;  /* 两端对齐 */
  gap: 20px;
}
.card {
  width: 200px;  /* 或使用 flex-basis 控制列数 */
}
```

当卡片数量为 5、7、10 等非整倍数时，最后一行会因为元素不足而被 `space-between` 强行拉开，出现下图中的尴尬布局：  

```text
[卡片1]    [卡片2]    [卡片3]
[卡片4]              [卡片5]   ← 两端分离，中间大片空白
```
这并非 Flexbox 的 Bug，而是 `space-between` 的预期行为。我们需要的是：最后一行保持与上方一致的左对齐。

**📊 方案对比总结**

| 方案 | 代码量 | 响应式 | 兼容性 | 推荐度 |
|------|--------|--------|--------|--------|
| **伪元素补位** | 少 | 需固定列数 | 现代浏览器 | ⭐⭐⭐ |
| **动态 margin-right** | 中 | 需配合媒体查询 | 极佳 | ⭐⭐⭐ |
| **空 div 占位** | 少 | 需 JS 配合 | 极佳 | ⭐ |
| **CSS Grid** | **极少** | **原生完美支持** | 现代浏览器 | ⭐⭐⭐⭐⭐ |
| **JS 动态补位** | 多 | 需监听 resize | 全部 | ⭐⭐ |


**本章小结**：Flexbox 将我们从浮动、定位和 `vertical-align` 的泥潭中解放出来。理解主轴与交叉轴是解锁所有属性的钥匙，`flex-grow` 和 `flex-shrink` 则赋予了你精细控制空间的权力。通过四个经典案例的练习，你已经掌握了现代网页布局的核心范式。在下一章中，我们将迈向二维布局的巅峰——CSS Grid。
