# 🗂️ 第6章：Grid —— 二维网格布局终极方案

如果说 Flexbox 是一维布局的瑞士军刀，那么 CSS Grid 就是二维布局的航空母舰。它允许你同时控制**行与列**，在画布上自由定义网格结构，然后将元素精确地放置到任意单元格中。本章将带你从网格的基础概念出发，掌握 `grid-template-areas` 的可视化魔力，领悟 `repeat()`、`minmax()` 等函数的自适应哲学，并最终用 Grid 实现圣杯布局与瀑布流。



## 📊 6.1 显式网格 vs 隐式网格：`grid-template-areas` 可视化命名区域

Grid 布局的核心是将容器划分为**行（Rows）** 和**列（Columns）**，形成单元格。你明确声明的行列构成了**显式网格（Explicit Grid）**；而当你放置的元素超出了显式网格范围，浏览器会自动生成新的行或列，这被称为**隐式网格（Implicit Grid）**。

### 🧩 创建显式网格：`grid-template-columns` 与 `grid-template-rows`

```html
<div class="explicit-grid">
  <div class="item">1</div>
  <div class="item">2</div>
  <div class="item">3</div>
  <div class="item">4</div>
  <div class="item">5</div>
  <div class="item">6</div>
</div>
```

```css
.explicit-grid {
  display: grid;
  grid-template-columns: 150px 1fr 2fr;  /* 三列：固定150px + 1份剩余 + 2份剩余 */
  grid-template-rows: 80px 120px;        /* 两行：80px 和 120px */
  gap: 10px;
  background: #f4f6f7;
  padding: 10px;
}
.item {
  background: #aed6f1;
  border: 1px solid #2980b9;
  display: flex;
  align-items: center;
  justify-content: center;
  font-weight: bold;
}
```

**效果**：6 个元素自动按先行后列的顺序填充 3×2 的网格。第 7 个元素若存在，将触发隐式网格。

### 🧬 隐式网格：`grid-auto-rows` 与 `grid-auto-columns`

当子项数量超过显式网格容量时，浏览器会根据 `grid-auto-rows` 和 `grid-auto-columns` 自动生成新行/列。

```css
.explicit-grid {
  grid-auto-rows: 60px;  /* 自动生成的行高度为 60px */
}
```

若放置了第 7 个元素，它会出现在第 3 行，高度为 60px。若不设置 `grid-auto-rows`，隐式行的高度将默认由内容撑开。

### 🎨 网格的终极可视化：`grid-template-areas`

`grid-template-areas` 允许你用**命名字符串**直接绘制网格布局，代码即设计图，极其直观。

**案例：一个典型的页面布局**

```html
<div class="page-layout">
  <header class="header">Header</header>
  <nav class="sidebar">Sidebar</nav>
  <main class="main">Main Content</main>
  <aside class="ads">Ads</aside>
  <footer class="footer">Footer</footer>
</div>
```

```css
.page-layout {
  display: grid;
  grid-template-columns: 200px 1fr 150px;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header header header"
    "sidebar main   ads"
    "footer footer footer";
  min-height: 100vh;
  gap: 15px;
  padding: 15px;
  background: #ecf0f1;
}
.header  { grid-area: header; background: #34495e; color: white; padding: 15px; }
.sidebar { grid-area: sidebar; background: #95a5a6; padding: 15px; }
.main    { grid-area: main; background: white; padding: 15px; }
.ads     { grid-area: ads; background: #95a5a6; padding: 15px; }
.footer  { grid-area: footer; background: #34495e; color: white; padding: 15px; }
```

**解读**：
- 第一行 `"header header header"` 表示 Header 占据三列。
- 第二行 `"sidebar main ads"` 将中间区域分为三部分。
- 第三行 `"footer footer footer"` 表示 Footer 横跨三列。

**修改布局极其简单**：只需调整 `grid-template-areas` 中的文字位置，无需改动 HTML 结构！例如，将 Ads 移到 Header 下方：

```css
grid-template-areas:
  "header header ads"
  "sidebar main   main"
  "footer footer footer";
```

这种**所见即所得**的布局声明方式，是 Grid 相比于浮动和定位的革命性进步。



## 🪄 6.2 神级函数：`repeat()`、`minmax()`、`fit-content()` 实现完美自适应

Grid 提供了一系列强大的函数，让你无需计算百分比就能创建弹性、响应式的网格。

### 1️⃣ `repeat()`：重复生成行列

```css
/* 生成 12 列等宽网格 */
grid-template-columns: repeat(12, 1fr);

/* 生成 3 列，模式为 100px 1fr 2fr */
grid-template-columns: repeat(3, 100px 1fr 2fr);  /* 实际产生 9 列 */

/* 尽可能多地生成至少 150px 的列（配合 minmax） */
grid-template-columns: repeat(auto-fill, minmax(150px, 1fr));
```

**`auto-fill` vs `auto-fit`**：
- `auto-fill`：尽可能多地创建轨道，即使没有内容填充，也会保留空轨道。
- `auto-fit`：将空轨道折叠，剩余空间由有内容的轨道均分。

**对比示例**：

```html
<div style="display: grid; grid-template-columns: repeat(auto-fill, minmax(120px, 1fr)); gap: 10px;">
  <div style="background: #aed6f1;">卡片1</div>
  <div style="background: #aed6f1;">卡片2</div>
</div>
```
- 若容器宽 500px，`auto-fill` 会创建 4 列（4×120=480），后两列为空，前两列各占 120px。
- 若换成 `auto-fit`，两列会均分 500px（各约 250px），空轨道被忽略。

### 2️⃣ `minmax(min, max)`：定义尺寸的弹性区间

`minmax()` 设置轨道的最小和最大尺寸，是实现响应式的核心。

```css
/* 侧边栏最小 200px，最大 300px；主内容区最小 400px，剩余空间均分 */
grid-template-columns: minmax(200px, 300px) minmax(400px, 1fr);
```

**经典应用：防止内容溢出**

```css
/* 避免内容撑破网格，结合 1fr 和 minmax */
grid-template-columns: 1fr minmax(0, 800px) 1fr;  /* 中间列最大 800px，且允许收缩到 0 */
```

`minmax(0, 1fr)` 是解决 Grid 子项内容溢出（如长英文单词）的关键技巧——将最小值设为 0，允许轨道收缩到足够小。

### 3️⃣ `fit-content(limit)`：根据内容自适应但有上限

```css
grid-template-columns: fit-content(300px) 1fr;
```

第一列宽度由内容决定，但不超过 300px。常用于标签或按钮列。

#### 4️⃣ `1fr` 的精确算法：`minmax(auto, 1fr)` 的陷阱

`1fr` 实际上是 `minmax(auto, 1fr)` 的简写。这意味着**轨道的最小值由内容的最小尺寸决定**。如果内容是一个超长的 URL 或连续英文字母，它会强制撑开轨道，破坏布局。

**修复**：使用 `minmax(0, 1fr)` 彻底允许轨道收缩。

```css
/* ❌ 长单词可能溢出 */
grid-template-columns: 1fr 1fr 1fr;

/* ✅ 强制允许收缩 */
grid-template-columns: repeat(3, minmax(0, 1fr));
```



## 🎯 6.3 对齐与间隙：Grid 中的对齐属性与 Flexbox 的细微差别

Grid 和 Flexbox 共享了许多对齐属性（如 `justify-content`、`align-items`），但作用域和行为存在关键差异。

### 📐 容器级别的对齐

| 属性 | 作用范围 | 说明 |
|------|----------|------|
| `justify-content` | 网格整体在容器内的水平对齐 | 当网格总宽度小于容器时生效 |
| `align-content` | 网格整体在容器内的垂直对齐 | 当网格总高度小于容器时生效 |
| `place-content` | `align-content` 和 `justify-content` 的简写 | `center stretch` |

### 📦 项目级别的对齐（作用于所有子项）

| 属性 | 作用范围 | 说明 |
|------|----------|------|
| `justify-items` | 子项在**其所在单元格**内的水平对齐 | 默认 `stretch`（填满单元格宽度） |
| `align-items` | 子项在**其所在单元格**内的垂直对齐 | 默认 `stretch`（填满单元格高度） |
| `place-items` | 简写 | `center center` 可实现完美居中 |

### 🧩 单个子项的对齐（覆盖 `*-items`）

| 属性 | 作用 |
|------|------|
| `justify-self` | 控制单个子项在单元格内的水平对齐 |
| `align-self` | 控制单个子项在单元格内的垂直对齐 |
| `place-self` | 简写 |

### 🔬 与 Flexbox 的细微差别

**差别一：`justify-content` 在 Grid 中作用于列轨道之间的空间分配，而在 Flex 中作用于主轴上的项目间隙。**

```css
.grid {
  display: grid;
  grid-template-columns: 100px 100px 100px;
  justify-content: space-between; /* 三列之间的剩余空间被均分到列与容器边缘 */
}
.flex {
  display: flex;
  justify-content: space-between; /* 三个项目之间的剩余空间均分 */
}
```

Grid 中，`space-between` 是在**轨道**之间分配空间；Flex 中是在**项目**之间分配空间。

**差别二：`margin: auto` 在 Grid 中的强大之处**

在 Flex 中，`margin: auto` 可以吸收剩余空间；在 Grid 中同样有效，且能实现更灵活的单项对齐。

```css
.grid-item {
  margin-left: auto;   /* 子项在单元格内右对齐 */
  margin-top: auto;    /* 子项在单元格内底对齐 */
}
```

这比 `justify-self` 和 `align-self` 更直观，因为它复用了盒模型的「自动边距」概念。

**差别三：`gap` 的行为完全一致**

无论是 Grid 还是 Flex，`gap`、`row-gap`、`column-gap` 的语义和效果完全相同。这是 CSS 工作组有意为之的设计统一。



## 🧱 6.4 实践模式：Holy Grail 响应式布局、瀑布流（Masonry）的 Grid 模拟法

### 1️⃣ 圣杯布局（Grid 版本）

Grid 实现圣杯布局比 Flex 更加简洁，因为行列可以同时定义。

```html
<div class="holy-grail-grid">
  <header>Header</header>
  <nav>Left Sidebar</nav>
  <main>Main Content</main>
  <aside>Right Sidebar</aside>
  <footer>Footer</footer>
</div>
```

```css
.holy-grail-grid {
  display: grid;
  grid-template-columns: 200px 1fr 180px;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header header header"
    "nav    main   aside"
    "footer footer footer";
  min-height: 100vh;
  gap: 10px;
  padding: 10px;
}
header { grid-area: header; background: #2c3e50; color: white; padding: 15px; }
nav    { grid-area: nav; background: #ecf0f1; padding: 15px; }
main   { grid-area: main; background: white; padding: 15px; }
aside  { grid-area: aside; background: #ecf0f1; padding: 15px; }
footer { grid-area: footer; background: #2c3e50; color: white; padding: 15px; }

/* 响应式：小于 768px 时变为单列 */
@media (max-width: 768px) {
  .holy-grail-grid {
    grid-template-columns: 1fr;
    grid-template-areas:
      "header"
      "nav"
      "main"
      "aside"
      "footer";
  }
}
```

**优势**：
- 无需嵌套额外的 Flex 容器，一层 Grid 搞定整体布局。
- 响应式只需重新定义 `grid-template-areas`，直观无比。

### 2️⃣ 瀑布流（Masonry）的 Grid 模拟

纯 CSS 瀑布流（高度不一的卡片紧密堆叠）目前没有标准属性，但可以借助 `grid` 和 `column` 方案模拟接近的效果。

**方案 A：CSS Grid 分列法（最稳健）**

```html
<div class="masonry-grid">
  <div class="masonry-item" style="height: 120px;">1</div>
  <div class="masonry-item" style="height: 180px;">2</div>
  <div class="masonry-item" style="height: 100px;">3</div>
  <div class="masonry-item" style="height: 200px;">4</div>
  <div class="masonry-item" style="height: 150px;">5</div>
  <div class="masonry-item" style="height: 130px;">6</div>
  <!-- ... -->
</div>
```

```css
.masonry-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
  gap: 16px;
  /* 关键：让每列内部的卡片自动紧密堆叠 */
  grid-auto-rows: 10px;  /* 设置一个很小的基准行高 */
}
.masonry-item {
  grid-row-end: span 15; /* 默认占 150px（15×10），然后由行内样式覆盖具体高度对应的 span */
  background: #d5e8d4;
  border-radius: 8px;
  padding: 10px;
  /* 需要通过 JS 或 CSS 变量动态计算 span 值，此处仅作演示 */
}
```

**局限性**：需要知道每张卡片的确切高度（或通过 JS 计算 `span` 值），并非真正的「自动紧密堆叠」。但这仍是目前生产环境中最常用的 Grid 模拟瀑布流方式。

**方案 B：CSS 多列（`column-count`）**

```css
.masonry-columns {
  column-count: 4;
  column-gap: 16px;
}
.masonry-columns .item {
  break-inside: avoid;  /* 防止卡片被截断到两列 */
  margin-bottom: 16px;
  background: #d5e8d4;
  border-radius: 8px;
  padding: 15px;
}
```

**效果**：卡片按**列优先**顺序排列（先填满第一列再填第二列），与典型瀑布流的「行优先」略有差异，但视觉上接近。

**方案 C：未来的原生 Grid Masonry（实验性）**

```css
.masonry-future {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  grid-template-rows: masonry;  /* 当前仅 Firefox 实验性支持 */
}
```

待 `masonry` 成为标准，瀑布流将迎来终极解法。

### 3️⃣ 自适应仪表盘（Dashboard）布局

```html
<div class="dashboard">
  <div class="card card1">统计卡片 1</div>
  <div class="card card2">统计卡片 2</div>
  <div class="card card3">统计卡片 3</div>
  <div class="card card4">图表区域（跨列）</div>
  <div class="card card5">活动列表</div>
</div>
```

```css
.dashboard {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  gap: 20px;
}
.card {
  background: white;
  padding: 20px;
  border-radius: 12px;
  box-shadow: 0 2px 8px rgba(0,0,0,0.1);
}
.card4 {
  grid-column: span 2;  /* 占两列 */
}
.card5 {
  grid-row: span 2;     /* 占两行 */
}

/* 响应式平板 */
@media (max-width: 992px) {
  .dashboard {
    grid-template-columns: repeat(2, 1fr);
  }
}
/* 响应式手机 */
@media (max-width: 576px) {
  .dashboard {
    grid-template-columns: 1fr;
  }
}
```

### 4️⃣ 重叠内容的艺术：`grid-area` 叠加

Grid 允许将多个元素放置到同一个单元格中，利用 `z-index` 控制层叠顺序，实现复杂的视觉效果（如文字覆盖图片）。

```html
<div class="hero">
  <img src="https://picsum.photos/800/400?random" alt="Hero" class="hero-img">
  <div class="hero-content">
    <h1>欢迎来到我的网站</h1>
    <p>探索更多精彩内容</p>
  </div>
</div>
```

```css
.hero {
  display: grid;
  grid-template: 1fr / 1fr;  /* 单行单列 */
}
.hero-img,
.hero-content {
  grid-area: 1 / 1 / 2 / 2;  /* 放入同一个单元格 */
}
.hero-img {
  width: 100%;
  height: 100%;
  object-fit: cover;
}
.hero-content {
  z-index: 1;
  color: white;
  background: rgba(0,0,0,0.4);
  display: flex;
  flex-direction: column;
  justify-content: center;
  align-items: center;
}
```

无需 `position: absolute`，无需 `top/left` 定位，Grid 原生支持元素重叠，且仍保留文档流的尺寸约束。


**本章小结**：CSS Grid 将二维布局的复杂度从「黑魔法」降维为「积木游戏」。通过显式网格与隐式网格的配合，`grid-template-areas` 的可视化声明，以及 `minmax()`、`repeat()` 等函数的加持，你可以用极少代码构建出健壮、响应式的复杂界面。掌握 Grid 与 Flexbox 的分工——**Grid 负责宏观页面骨架，Flex 负责微观组件内部**——你将拥有现代 CSS 布局的完整心法。
