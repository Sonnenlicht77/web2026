# 🗺️ 第7章：定位与堆叠 —— Z 轴上的世界

当 Flexbox 和 Grid 统治了二维平面，`position` 则掌控着第三个维度——Z 轴。它让元素能够脱离标准流，悬浮、固定、重叠，创造出模态框、悬浮导航、下拉菜单等交互界面。然而，定位的「自由」也伴随着「失控」：元素莫名其妙地错位、`z-index` 设得再大也不生效。本章将带你彻底征服这片垂直疆域。



## 📌 7.1 五种 `position` 值的行为全解

CSS 提供了五种 `position` 值，每种都有独特的定位参照系和文档流行为。

### 1️⃣ `static`：默认值，安分守己的文档流公民

- **行为**：元素按标准流排列，忽略 `top`、`right`、`bottom`、`left` 和 `z-index`。
- **适用场景**：99% 的元素都应保持 `static`，除非需要特殊定位。

```css
.static-box {
  position: static;  /* 默认值，通常无需显式声明 */
}
```

### 2️⃣ `relative`：相对于自身的偏移，保留原位占位

- **参照系**：元素**原本在文档流中的位置**。
- **关键特性**：**元素偏移后，它在文档流中的原始空间仍然保留**，不会影响其他元素布局。
- **典型用途**：
  - 作为 `absolute` 子元素的定位容器（创建「定位上下文」）。
  - 微调元素位置而不破坏整体布局。

```html
<div class="container">
  <div class="box">正常盒子</div>
  <div class="box relative-box">相对定位盒子</div>
  <div class="box">正常盒子</div>
</div>
```

```css
.box {
  width: 100px;
  height: 50px;
  background: #aed6f1;
  border: 1px solid #2980b9;
}
.relative-box {
  position: relative;
  top: 20px;
  left: 30px;
  background: #f9e79f;
}
```

**效果**：中间的黄色盒子向右偏移 30px，向下偏移 20px，但它原本占据的空间依然存在（一个「空位」），第三个盒子并没有上移填补。

### 3️⃣ `absolute`：相对于最近的非 `static` 祖先定位，完全脱离文档流

- **参照系**：向上查找第一个 `position` 不为 `static` 的祖先元素（通常是 `relative`）。若没有，则相对于**初始包含块**（通常是 `<html>` 或视口）。
- **关键特性**：元素**彻底脱离文档流**，不占据任何空间，后续元素会当作它不存在。
- **典型用途**：下拉菜单、弹窗、角标、悬浮按钮。

```html
<div class="card" style="position: relative; width: 300px; height: 150px; background: #ecf0f1;">
  <span class="badge" style="position: absolute; top: -10px; right: -10px;">HOT</span>
  <p>卡片内容，左上角有角标飘出。</p>
</div>
```

```css
.badge {
  background: #e74c3c;
  color: white;
  padding: 4px 8px;
  border-radius: 20px;
  font-size: 12px;
}
```

**效果**：`HOT` 角标悬浮在卡片右上角外侧，不占用卡片内容空间。

**尺寸特性**：绝对定位元素的宽度默认由内容撑开（`width: auto` 行为变为「收缩到合适」），除非显式设置 `width` 或同时设置 `left` 和 `right` 拉伸。

```css
.stretch-absolute {
  position: absolute;
  left: 10px;
  right: 10px;
  /* 不设置 width，元素宽度 = 父容器宽度 - 20px */
}
```

### 4️⃣ `fixed`：相对于视口定位，滚动也无法撼动

- **参照系**：**浏览器视口（Viewport）**。
- **关键特性**：脱离文档流，且页面滚动时位置固定不变。
- **典型用途**：固定导航栏、返回顶部按钮、客服悬浮窗。

```html
<button class="back-to-top" style="position: fixed; bottom: 30px; right: 30px;">⬆️ 顶部</button>
```

**特殊情况**：当祖先元素设置了 `transform`、`perspective`、`filter` 等属性时，`fixed` 的参照系会**变为该祖先元素**，而非视口。这是常被忽略的陷阱。

```html
<div style="transform: translateX(0);">
  <div style="position: fixed; top: 0; left: 0;">我并不是相对于视口固定！</div>
</div>
```

### 5️⃣ `sticky`：滚动到阈值时「粘」住，相对与固定的混合体

- **行为**：元素在滚动跨越特定阈值前表现为 `relative`，跨越后表现为 `fixed`（在父容器内）。
- **关键约束**：
  - 必须指定 `top`、`right`、`bottom`、`left` 中的至少一个值，作为「粘住」的阈值。
  - 粘性效果**仅在父容器范围内有效**，一旦父容器滚出视口，`sticky` 元素也随之滚走。

```html
<div style="height: 200px; overflow-y: auto; border: 1px solid #ccc;">
  <div style="height: 50px; background: #ddd;">滚动区域头部</div>
  <div class="sticky-header" style="position: sticky; top: 0; background: #f9e79f; padding: 10px;">
    粘性标题（滚动到顶部时固定）
  </div>
  <div style="height: 400px; background: linear-gradient(#fff, #eee);">
    <p>大量内容...</p>
  </div>
</div>
```

**效果**：当向下滚动时，标题栏在触及滚动容器顶部后会「粘」住，后续内容在其下方继续滚动；当父容器完全滚出视口，标题随之消失。



## 🐞 7.2 粘性定位（Sticky）失效诊断清单

`position: sticky` 是失效频率最高的 CSS 属性之一。如果你发现它「粘不住」，请按以下清单逐项排查：

### 🧾 失效原因一：未指定 `top` / `left` 等阈值

```css
/* ❌ 无效 */
.sticky {
  position: sticky;
  /* 缺少 top/left 等，浏览器不知道何时粘住 */
}

/* ✅ 有效 */
.sticky {
  position: sticky;
  top: 0;
}
```

### 🧾 失效原因二：祖先元素设置了 `overflow: hidden` / `auto` / `scroll` / `overlay`

这是最隐蔽的杀手。**如果 `sticky` 元素的任意祖先元素（除根元素外）设置了 `overflow` 非 `visible`，且该祖先不是滚动容器本身，则 `sticky` 失效。**

```html
<!-- ❌ 失效 -->
<div style="overflow: hidden; height: 300px;">
  <div style="overflow: auto; height: 200px;">
    <div style="position: sticky; top: 0;">粘不住！</div>
  </div>
</div>

<!-- ✅ 有效 -->
<div style="height: 300px;">
  <div style="overflow: auto; height: 200px;">
    <div style="position: sticky; top: 0;">能粘住</div>
  </div>
</div>
```

**原理解释**：`overflow` 非 `visible` 会创建一个新的滚动边界，干扰 `sticky` 的参照系计算。最外层的 `overflow: hidden` 阻断了 `sticky` 与视口的联系。

### 🧾 失效原因三：父容器高度与 `sticky` 元素高度相同或没有滚动空间

`sticky` 元素只能在父容器内部「滑动」。如果父容器高度等于 `sticky` 元素高度，或者内容太少不足以触发滚动，自然看不到粘性效果。

```html
<!-- ❌ 父容器与子元素同高，无法滚动 -->
<div style="height: 50px; overflow: auto;">
  <div style="position: sticky; top: 0; height: 50px;">不会滚动</div>
</div>

<!-- ✅ 父容器有足够高度产生滚动 -->
<div style="height: 200px; overflow: auto;">
  <div style="position: sticky; top: 0;">粘性头部</div>
  <div style="height: 400px;">很多内容...</div>
</div>
```

### 🧾 失效原因四：`sticky` 元素是 Flex 或 Grid 子项时的对齐影响

某些情况下，Flex/Grid 容器的 `align-items` 可能拉伸子项，导致 `sticky` 失效。通常给 `sticky` 元素设置 `align-self: flex-start` 可解决。

```css
.flex-container {
  display: flex;
  align-items: stretch;  /* 默认 */
}
.sticky-child {
  position: sticky;
  top: 0;
  align-self: flex-start;  /* 防止拉伸导致失效 */
}
```

### 🧾 失效原因五：浏览器兼容性或特殊场景

- Safari 旧版本对 `sticky` 支持有缺陷，尤其是结合 `thead` 或 `display: table-*` 时。
- 若 `sticky` 元素本身是 `display: inline`，可能表现异常。

### 📋 快速自检清单（打印或截图保存）

- [ ] 是否设置了 `top` / `right` / `bottom` / `left` 至少一个？
- [ ] 所有祖先元素（除滚动容器外）的 `overflow` 是否均为 `visible`？
- [ ] 滚动容器的高度是否大于 `sticky` 元素的高度，且有足够内容产生滚动？
- [ ] 父容器是否是 Flex/Grid 容器？若是，是否设置了 `align-self`？
- [ ] 浏览器是否为 Safari？是否涉及表格元素？



## 🎚️ 7.3 层叠上下文（Stacking Context）：`z-index` 失效的根本原因图解

`z-index` 似乎很简单：数值越大越靠上。但实际开发中常遇到「`z-index: 9999` 依然被覆盖」的诡异现象。问题的根源在于**层叠上下文**。

### 🧩 什么是层叠上下文？

层叠上下文是一个三维概念，可以理解为页面上的独立「图层」。每个层叠上下文内部有一套独立的 `z-index` 排序体系，**不同层叠上下文之间的元素无法通过 `z-index` 直接比较**，只能比较层叠上下文本身的层叠顺序。

### 🔨 创建层叠上下文的常见触发方式

| 触发条件 | 示例 |
|---------|------|
| 根元素 `<html>` | 天然层叠上下文 |
| `position` 为 `absolute` / `relative` 且 `z-index` 不为 `auto` | `position: relative; z-index: 1;` |
| `position` 为 `fixed` / `sticky` | 无论 `z-index` 多少，始终创建 |
| `opacity` 小于 1 | `opacity: 0.99;` |
| `transform` 不为 `none` | `transform: translateX(0);` |
| `filter` 不为 `none` | `filter: blur(0);` |
| `will-change` 指定上述属性 | `will-change: transform;` |
| `isolation: isolate` | **专门用于创建层叠上下文，无副作用** |

### 🧪 经典案例：为什么弹窗的 `z-index: 9999` 依然被覆盖？

```html
<div class="parent1" style="position: relative; z-index: 1;">
  <div class="modal" style="position: fixed; z-index: 9999;">弹窗</div>
</div>
<div class="parent2" style="position: relative; z-index: 2;">
  <div class="overlay" style="position: fixed; z-index: 10;">遮罩</div>
</div>
```

**层叠关系分析**：
- `parent1` 创建层叠上下文，层叠顺序为 1。
- `parent2` 创建层叠上下文，层叠顺序为 2。
- `modal` 虽然是 `fixed` 定位，`z-index: 9999`，但它**被包裹在层叠顺序为 1 的 `parent1` 中**。
- `overlay` 的 `z-index` 仅为 10，但它属于层叠顺序为 2 的 `parent2`。
- **结论**：`parent2` 整体高于 `parent1`，所以 `overlay` 覆盖 `modal`，`9999` 毫无作用。

### 📊 层叠顺序图解（从下到上）

在同一层叠上下文内部，元素按以下顺序堆叠（编号越大越靠上）：

1. 层叠上下文的背景和边框
2. `z-index` 为负值的定位元素
3. 标准流中的块级盒子、浮动盒子
4. 标准流中的行内盒子
5. `z-index: auto` 或 `z-index: 0` 的定位元素
6. `z-index` 为正值的定位元素（数值越大越靠上）

#### ✅ 解决方案：将弹窗提升到更高层级的层叠上下文，或使用 `isolation: isolate`

**方案一：将弹窗移到根元素下（如 `<body>` 的直接子元素）**

```html
<!-- 使用 Portal 或直接挂载到 body -->
<body>
  <div id="app">...</div>
  <div class="modal" style="position: fixed; z-index: 1000;">弹窗</div>
</body>
```

**方案二：主动创建新的层叠上下文并提高其层级**

```css
.modal-container {
  position: relative;
  z-index: 999;  /* 确保其层叠上下文足够高 */
}
```

**方案三：使用 `isolation: isolate` 隔离，并配合 `z-index`**

```css
.modal-wrapper {
  isolation: isolate;  /* 创建新层叠上下文，不依赖 z-index */
}
```

### 🔍 DevTools 调试技巧：可视化层叠上下文

Chrome DevTools 的 **Layers** 面板可以查看页面的图层构成。
1. 打开 DevTools → 更多工具 → Layers。
2. 选中元素，查看其所在的合成层及层叠原因。
3. 或者在 **Rendering** 标签中勾选 **Layer borders**，用不同颜色的边框区分图层。



## 🧭 7.4 总结：什么时候该用定位，什么时候该用 Grid？

定位和现代布局（Flex/Grid）并非竞争关系，而是互补关系。理解各自的使用场景，才能写出干净、可维护的 CSS。

### ✅ 优先使用 Grid / Flex 的场景

- **页面整体骨架**（圣杯、双飞翼、仪表盘）→ **Grid**
- **一维排列**（导航菜单、卡片列表、按钮组）→ **Flex**
- **响应式网格**（相册、商品列表）→ **Grid + `auto-fit`**
- **垂直居中、等高列** → **Flex / Grid**（远比定位简单）
- **任何可以用文档流描述的关系** → 保持文档流

### ✅ 必须使用定位的场景

- **元素需要悬浮于其他内容之上**（模态框、下拉菜单、Tooltip、通知徽章）→ `absolute` / `fixed`
- **元素需要「脱离」标准流，不占用空间**（角标、覆盖层、加载蒙层）→ `absolute`
- **元素需要固定在视口**（返回顶部、客服按钮、固定导航）→ `fixed`
- **滚动吸顶效果**（表格标题、分类栏）→ `sticky`
- **创建 `absolute` 子元素的定位参照容器** → 父元素 `relative`

#### ⚖️ 决策流程图（文字版）

```
需要元素脱离标准流，悬浮在其他元素之上？
├── 是 → 是否需要相对于视口固定？
│   ├── 是 → position: fixed
│   └── 否 → 是否需要滚动到某位置后「粘住」？
│       ├── 是 → position: sticky
│       └── 否 → position: absolute（记得给父容器 position: relative）
└── 否 → 是否只需要微调位置，但保留原位占位？
    ├── 是 → position: relative + top/left
    └── 否 → 使用 Flexbox 或 Grid 解决布局问题
```

### 💎 最佳实践忠告

1. **定位是「覆盖层」工具，不是「布局」工具。** 不要用 `absolute` 来做三列布局。
2. **`relative` 最大的价值是创建定位上下文，而不是频繁移动元素。** 大量使用 `top/left` 微调通常是设计稿或 HTML 结构有问题的信号。
3. **`z-index` 的管理需要系统化。** 建议在项目中定义 `z-index` 变量（如 `--z-dropdown: 100; --z-modal: 1000`），避免随意写 `9999`。
4. **排查 `z-index` 失效时，先查层叠上下文。** 90% 的问题出在某个祖先元素悄悄创建了层叠上下文。



**本章小结**：定位赋予了元素挣脱文档流的自由，但自由需要约束。理解五种 `position` 的行为边界，警惕 `sticky` 的失效陷阱，洞察层叠上下文的隔离法则，你便能在 Z 轴上挥洒自如。记住：Grid 与 Flex 构建世界，定位则负责点缀星空。
